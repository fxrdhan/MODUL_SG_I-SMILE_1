A. Pengantar

Pada modul ini, kita akan belajar tentang Teachable Machine, platform yang memungkinkan kita untuk melatih mesin dalam mengenali objek, suara, atau gerakan. Teachable Machine adalah cara yang baik untuk memahami konsep dasar Machine Learning dan computer vision, dan kita akan memiliki kesempatan untuk membuat model sederhana yang dapat mengenali pola-pola yang kita tentukan.

B. Tujuan

  1. Memahami konsep dasar Machine Learning dan pengenalan objek, suara, maupun gerakan.
  1. Mempelajari cara menggunakan Teachable Machine untuk melatih model dasar.
  1. Membuat model yang dapat mengenali objek atau suara yang ditentukan.

C. Alat

  1. PC / Laptop
  1. Koneksi Internet
  1. Webcam dan Microphone

D. Materi Pembelajaran

1. Machine Learning

Machine Learning (Pembelajaran Mesin) adalah cabang dari kecerdasan buatan yang mempelajari metode/algoritma agar komputer dapat belajar dari data (pengalaman) untuk menemukan pola, membuat prediksi, atau mengambil keputusan tanpa diprogram aturan-aturan spesifik untuk setiap kasus. Konsep ini banyak digunakan untuk masalah berbasis data seperti klasifikasi, regresi, dan penemuan struktur data.

Terdapat beberapa jenis utama Machine Learning:

  • Supervised Learning: Model dilatih menggunakan data berlabel, artinya setiap contoh pada data pelatihan memiliki “jawaban benar” berupa target/label. Tujuannya adalah mempelajari pemetaan dari input ke output untuk melakukan prediksi pada data baru, misalnya klasifikasi dan regresi.
  • Unsupervised Learning: Model mempelajari data tanpa label untuk menemukan pola/struktur tersembunyi. Pendekatan ini sering digunakan untuk clustering, reduksi dimensi, atau pemodelan distribusi data.
  • Reinforcement Learning: Agen belajar melalui interaksi dengan lingkungan. Agen memilih aksi, lalu menerima umpan balik berupa reward (hadiah) atau penalty (hukuman). Tujuannya adalah mempelajari strategi (policy) yang memaksimalkan total reward dalam jangka panjang.

2. Teachable Machine

Teachable Machine adalah alat berbasis web dari Google Creative Lab yang membantu pengguna membuat model machine learning sederhana dengan cepat tanpa perlu menulis kode. Pengguna menyiapkan beberapa contoh untuk tiap kategori (kelas), misalnya gambar, suara, atau pose yang selanjutnya alat ini melatih model agar mampu mengenali input baru berdasarkan contoh yang diberikan.

Dalam praktik pembelajaran, Teachable Machine sering dipakai untuk memahami Supervised Learning karena pengguna biasanya memberi contoh per kelas seperti halnya kucing dengan anjing sehingga model belajar membedakan kelas-kelas tersebut saat melakukan prediksi.

Teachable Machine mendukung pelatihan model untuk gambar, suara, dan pose. Hasil model dapat digunakan kembali untuk proyek seperti situs web atau aplikasi (misalnya untuk prototipe pembelajaran dan eksperimen)

Teachable Machine mendukung tiga jenis pengenalan utama:

  1. Pengenalan Gambar: Mesin dapat dilatih untuk mengenali objek atau pola visual tertentu di dalam gambar. Contohnya pada gambar di bawah ini, mesin telah dilatih menggunakan sampel webcam untuk mengenali dan membedakan gestur tangan pada permainan Batu, Gunting, Kertas (rock, paper, scissors). Saat pengguna menunjukkan gestur gunting di depan kamera, mesin berhasil mendeteksinya dengan akurasi 100% pada kelas scissors.
  1. Pengenalan Suara: Mesin dapat dilatih untuk mengenali berbagai jenis suara manusia atau lingkungan. Dalam contoh di bawah, data latih (train) berupa suara background noise, clapping, and snap.
  1. Pengenalan Gerakan: Memungkinkan pengenalan gerakan atau postur tubuh dengan kamera webcam. Contohnya membedakan gerakan berdiri (Standing) dan duduk (Sitting).

E. Memulai Project Teachable Machine

Langkah Persiapan Awal:
Buka browser web (Google Chrome atau Edge). Kunjungi situs web Teachable Machine di alamat: https://teachablemachine.withgoogle.com/. Kemudian klik tombol "Get Started".

Setelah itu, Anda akan dihadapkan pada 3 pilihan tipe proyek. Mari kita coba satu per satu.

E.1. Pengenalan Gambar

Latar Belakang

Permainan Batu Gunting Kertas (Rock Paper Scissors) adalah contoh klasik yang sangat cocok untuk mempraktikkan computer vision karena:

  • Memiliki jumlah kelas yang jelas dan terbatas (3 kelas).
  • Perbedaan visual antar gestur cukup signifikan sehingga model lebih mudah belajar.

Dalam praktik ini, komputer akan "melihat" tangan kita melalui webcam dan menentukan apakah gestur yang ditampilkan adalah Batu, Gunting, atau Kertas.

Langkah-Langkah Praktik

  1. Pada menu pilihan Project, klik Image Project.
  1. Selanjutnya akan diberikan dua opsi. Pilih Standard Image Model.
    (Catatan: Opsi "Embedded Image Model" digunakan jika kita ingin menanamkan model pada mikrokontroler dengan memori terbatas. Untuk study group modul pertama ini, Standard Model dipilih).
  1. Secara default, Teachable Machine hanya menyediakan dua kelas. Karena kita membutuhkan tiga kelas, kita perlu menambahkan satu kelas lagi. Ubah nama kelas menjadi:
    • "Class 1" untuk "Rock" (Batu)
    • "Class 2" untuk "Scissors" (Gunting)
    • Klik "Add a class" untuk beri nama "Paper" (Kertas)

    Ketiga label ini adalah kategori yang akan dipelajari oleh komputer.

  1. Mengumpulkan Data Latih — Kelas Batu (Rock)

    Pada kelas Rock, klik ikon Webcam.

    Setelah diklik, tampilan antarmuka webcam akan muncul seperti gambar berikut.

    Sebelum merekam, kita perlu mengatur pengaturan terlebih dahulu. Klik ikon gerigi (⚙️ Settings) yang berada di sebelah kanan tombol "Hold to Record". Akan muncul panel pengaturan seperti gambar berikut.

    Lakukan pengaturan berikut pada panel Settings:

    • Hold to Record: Secara default bernilai ON — ubah menjadi OFF agar perekaman berjalan otomatis tanpa harus menahan tombol.
    • Duration: Isi dengan 10 detik. Dengan FPS sebesar 24 sampel per detik, dalam 10 detik akan diperoleh 200+ sampel gambar.

    Setelah selesai, klik tombol "Save Settings" untuk menyimpan konfigurasi.

    Selanjutnya, klik tombol "Record" (nama tombol kini berubah menjadi "Record 10 Seconds" sesuai durasi yang telah diatur). Sebelum merekam, pastikan:

    • Izin kamera telah diberikan pada browser Anda.
    • Webcam sudah menyala dan menampilkan gambar.

    Setelah tombol "Record 10 Seconds" diklik, perekaman belum langsung dimulai dan akan ada hitungan mundur 5 detik terlebih dahulu. Gunakan waktu ini untuk mempersiapkan posisi tangan Anda.

    Selama perekaman berlangsung, gerakkan tangan yang mengepal ke berbagai arah secara perlahan dan bervariasi, seperti:

    • Miringkan kepalan ke kiri, kanan, atas, dan bawah.
    • Dekatkan dan jauhkan tangan dari kamera.
    • Putar pergelangan tangan perlahan agar model mengenal gestur dari berbagai sudut pandang.

    Setelah perekaman selesai, Anda akan melihat thumbnails sampel gambar yang berhasil dikumpulkan muncul di panel kanan. Pastikan jumlah sampel sudah mencapai 200 sampel atau lebih.

  1. Mengumpulkan Data Latih — Kelas Gunting (Scissors)

    Bentuk gestur Gunting dengan mengacungkan jari telunjuk dan jari tengah membentuk huruf "V", sementara jari lainnya dilipat.

    Klik kelas Scissors, lalu klik Webcam. Ulangi langkah yang sama seperti pada kelas Rock lalu klik "Record 10 Seconds". Variasikan sudut, jarak, dan orientasi tangan selama perekaman. Kumpulkan 200+ sampel.

  1. Mengumpulkan Data Latih — Kelas Kertas (Paper)

    Bentuk gestur Kertas dengan membuka semua jari tangan selebar mungkin dan merentangkannya menghadap ke kamera.

    Klik kelas Paper, lalu klik Webcam. Ulangi kembali langkah yang sama lalu klik "Record 10 Seconds". Gerakkan tangan terbuka ke berbagai arah dan jarak selama perekaman. Kumpulkan 200+ sampel.

  1. Setelah data latih dari ketiga kelas (Rock, Scissors, Paper) terkumpul, klik tombol "Train Model" di panel kanan.

    Proses ini akan memulai fase transfer learning — yaitu teknik di mana model yang sudah dilatih sebelumnya (oleh Google) pada jutaan gambar digunakan sebagai dasar, lalu disesuaikan (fine-tuned) dengan sampel kelas yang baru saja kita kumpulkan. Karena itulah pelatihan di Teachable Machine berlangsung sangat cepat meskipun sampel kita terbatas.

    Tunggu hingga proses pelatihan selesai. Selama proses pelatihan, pastikan untuk tidak membuka/pindah ke tab browser lain karena dapat mengganggu proses.

  1. Menguji Model

    Setelah pelatihan selesai, model langsung dapat diuji pada panel Preview di sebelah kanan. Coba tampilkan masing-masing gestur tangan ke kamera:

    • Tunjukkan gestur Batu lalu perhatikan bar persentase pada Output, seharusnya tinggi di kelas "Rock".
    • Tunjukkan gestur Gunting dan seharusnya tinggi di kelas "Scissors".
    • Tunjukkan gestur Kertas dan seharusnya tinggi di kelas "Paper".

    Jika semua gestur terdeteksi dengan akurasi berada di rentang angka 90-100%, berarti model telah belajar dengan baik!

E.2. Pengenalan Suara

  1. Kembali ke halaman New Project, kali ini pilih Audio Project.
  1. Dalam kegiatan ini, kita akan membuat model yang mengenali dua kalimat, yaitu: "Hidup Jokowi" dan "Antek asing". Namun untuk audio, wajib ada satu kelas tambahan bawaan yaitu Background Noise (suara gangguan dalam ruangan).
  1. Pada Class pertama (Background Noise), klik Mic. Tekan "Record 20 Seconds" lalu diamkan mikrofon merekam kebisingan ruangan. Setelah selesai, klik "Extract Sample".

    Setelah perekaman selesai, klik pada tombol “Extract Sample” untuk mengekstrak audio yang telah direkam menjadi 20 audio terpotong.

    Sekarang Anda akan melihat hasil adio terpotong yang sudah siap dipakai.

  1. Ubah "Class 2" menjadi "Hidup Jokowi". Klik Mic. Pada ikon gerigi (Settings), Anda bisa mengubah durasi rekam menjadi 20 detik. Tekan "Record", lalu ucapkan "Hidup Jokowi" berulang kali dengan nada berbeda. Setelah selesai, klik Extract Sample.
  1. Tambahkan kelas baru (Add a class), beri nama "Antek asing". Ulangi proses yang sama: Rekam selama 10 detik sambil mengucapkan "Antek asing" berulang kali, lalu ekstrak sampelnya.
  1. Klik Train Model. Ingat, jangan membuka tab lain selama proses training berlangsung.
  1. Lakukan uji coba (Preview). Bicaralah ke depan microphone Anda dan lihat apakah bar persentase pada Output menunjukkan angka yang tinggi sesuai dengan kalimat yang Anda sebutkan. Dalam contoh ini, akurasi pengenalan suara berhasil menangkap dengan baik pada kelas “Hidup Jokowi”.

E.3. Pengenalan Gerakan

  1. Kembali ke halaman awal New Project, lalu pilih Pose Project.
  1. Kita akan menggunakan tiga kelas pose, yaitu: "Neutral", "Iziinnn", dan "Tidak". Kelas Neutral berfungsi sebagai kelas penampung untuk pose biasa/berdiri tegap, sehingga model tidak bias ketika postur tubuh tidak termasuk ke dalam kelas manapun — mirip seperti kelas Background Noise pada pengenalan suara.

    Ubah nama kelas 1, kelas 2, dan tambahkan kelas ketiga sesuai nama gerakan tersebut:

    • "Class 1" → "Neutral"
    • "Class 2" → "Iziinnn"
    • Klik "Add a class" → beri nama "Tidak Iziinnn"
  1. Pada kelas Neutral, klik Webcam. Mundur sedikit agar seluruh postur tubuh Anda terlihat. Berdiri tegap dalam posisi normal (tanpa gerakan khusus). Tekan tombol "Record 10 Seconds" sambil mengubah sudut pandang (Point of View/POV) secara perlahan — misalnya sedikit miring ke kiri, kanan, maju, atau mundur. Kumpulkan sekitar 200+ pose sample.
  1. Pada kelas Iziinnn, klik Webcam. Mundur sedikit agar postur tangan dan wajah Anda terlihat. Tekan tombol "Record 10 Seconds" sambil melakukan gestur izin. Kumpulkan sekitar 200+ pose sample.
  1. Pada kelas Tidak Iziinnn, rekam menggunakan Webcam dengan menekan "Record 10 Seconds" sambil melambaikan tangan ke arah kamera hingga terkumpul 200+ sampel.
  1. Klik Train Model dan tunggu hingga prosesnya selesai.
  1. Pada panel Preview, gerakkan tubuh Anda sesuai dengan kelas yang dibuat dan perhatikan bilah Output. Pastikan:
    • Saat berdiri tegap biasa → bar Neutral yang tinggi.
    • Saat melakukan gestur Iziinnn → bar Iziinnn yang tinggi.
    • Saat melambaikan tangan → bar Tidak Iziinnn yang tinggi.

    Jika model mengenali ketiga kondisi dengan akurasi mendekati 100%, maka model telah bekerja dengan baik.

F. Menyimpan & Membagikan Model (Export Model)
Model yang sudah selesai dilatih dan diuji tidak hanya berhenti di website ini saja. Anda dapat mengekspornya untuk digunakan pada proyek coding sungguhan (misalnya membuat website atau aplikasi smartphone berbasis AI).

  1. Pada panel Preview, perhatikan tombol di bagian atas dan klik Export Model.
  1. Akan muncul sebuah pop-up. Anda memiliki beberapa pilihan:
    • Upload (Shareable link): Model Anda akan diunggah ke server cloud Google, dan Anda akan mendapatkan link yang bisa dibagikan atau disisipkan ke dalam code (HTML/JavaScript).
    • Download: Anda dapat mengunduh model mentahnya dalam bentuk Tensorflow.js (untuk web) atau Tensorflow Lite (untuk Android/Mikrokontroler).

G. Fitur Tambahan (Hyperparameter)
Pada bagian Training, jika Anda menekan menu Advanced, Anda akan menemukan beberapa pengaturan lanjutan (hyperparameter) yang bisa diubah untuk meningkatkan akurasi model:

  1. Epochs
    Satu epoch berarti bahwa setiap sampel dalam dataset pelatihan telah melewati model pelatihan setidaknya satu kali. Jika kita mengatur epochs menjadi 50, ini berarti model yang kita latih akan membaca ulang seluruh dataset pelatihan sebanyak 50 kali. Jumlah epoch memengaruhi seberapa baik model kita dapat belajar dari data. Semakin besar jumlahnya, semakin baik model belajar memprediksi. Namun, jika terlalu banyak, dapat mengakibatkan overfitting (model hanya hafal data latih, tapi gagal mengenali data baru di dunia nyata).
  1. Batch Size
    Batch adalah sejumlah sampel yang digunakan dalam satu iterasi pelatihan. Misalnya, jika kita memiliki 80 gambar dan memilih batch size 16, data akan dibagi menjadi (80 / 16) = 5 batch. Batch size yang lebih besar dapat mempercepat pelatihan, tetapi memerlukan lebih banyak memori komputer.
  1. Learning Rate
    Learning rate adalah parameter yang menentukan seberapa "besar langkah" model dalam belajar ketika menyesuaikan diri selama pelatihan. Jika learning rate terlalu besar, model dapat melewatkan titik minimum (titik akurasi terbaik). Jika terlalu kecil, pelatihan bisa berjalan sangat lambat.
